# 多模式脱敏系统设计

## 模式对比

| 模式 | 配置对象 | 源表 | 目标表 | 角色 | 适用场景 |
|------|----------|------|--------|------|----------|
| STATIC | 任务 | ✓ | ✓ | - | 开发测试、数据导出 |
| DYNAMIC | 表 | ✓ | - | ✓ | 生产环境权限控制 |
| ANONYMIZE | 表 | ✓ | - | - | GDPR合规、数据销毁 |
| GENERALIZE | 任务 | ✓ | ✓ | - | 数据分析、统计报表 |

## 动态脱敏配置设计

动态脱敏是针对**表**进行配置，而非任务：

```
┌─────────────────────────────────────────────────────────────┐
│                    动态脱敏配置                              │
├─────────────────────────────────────────────────────────────┤
│  数据源: production_db                                      │
│  Schema: public                                             │
│  表: users                                                  │
├─────────────────────────────────────────────────────────────┤
│  字段脱敏规则:                                              │
│  - email: anon.partial_email()                              │
│  - phone: anon.partial(3, '*', 4)                           │
│  - name: anon.fake_first_name()                             │
├─────────────────────────────────────────────────────────────┤
│  被脱敏角色: analyst, developer                             │
│  豁免角色: admin, dba                                       │
└─────────────────────────────────────────────────────────────┘
```

执行后：
1. 为表字段设置 SECURITY LABEL
2. 创建脱敏视图（可选）
3. 授权指定角色访问

## 静态脱敏配置设计

静态脱敏是针对**任务**进行配置：

```
┌─────────────────────────────────────────────────────────────┐
│                    静态脱敏任务                              │
├─────────────────────────────────────────────────────────────┤
│  任务名称: 测试环境数据准备                                  │
│  数据源: production_db                                      │
│  源Schema: public                                           │
│  目标Schema: test_env                                       │
├─────────────────────────────────────────────────────────────┤
│  表配置:                                                    │
│  - users → users_masked                                     │
│    - email: anon.fake_email()                               │
│    - phone: anon.partial(3, '*', 4)                         │
│  - orders → orders_masked                                   │
│    - customer_name: anon.fake_name()                        │
└─────────────────────────────────────────────────────────────┘
```

执行后：
1. 创建目标表
2. 复制数据
3. 应用脱敏规则

## 原地匿名化配置设计

原地匿名化是针对**表**进行配置，直接修改原表：

```
┌─────────────────────────────────────────────────────────────┐
│                    原地匿名化                                │
├─────────────────────────────────────────────────────────────┤
│  数据源: production_db                                      │
│  Schema: public                                             │
│  表: customers                                              │
├─────────────────────────────────────────────────────────────┤
│  ⚠️ 警告: 此操作将永久修改原表数据，不可恢复！              │
├─────────────────────────────────────────────────────────────┤
│  字段脱敏规则:                                              │
│  - email: anon.fake_email()                                 │
│  - phone: anon.partial(3, '*', 4)                           │
│  - id_card: anon.hash()                                     │
└─────────────────────────────────────────────────────────────┘
```

执行后：
1. 备份原表（可选）
2. UPDATE 原表数据
