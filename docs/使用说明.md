## 快速开始

### 1. 启动系统

#### 使用 Docker（推荐）
```bash
# 克隆项目
git clone <repository-url>
cd data-masking-tool

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

#### 手动启动
```bash
# 后端
cd backend
python3.10 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 配置数据库
python scripts/init_db.py
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm install
cp .env.example .env
npm run dev
```

### 2. 访问系统

启动完成后，通过以下地址访问系统：
- **前端界面**: http://localhost（开发模式: http://localhost:5173）
- **后端API**: http://localhost:8000
- **API文档**: http://localhost:8000/api/docs

### 3. 默认账号

|--------|------|------|
| admin | admin123 | 超级管理员 |

## 数据脱敏

### 脱敏模式说明

系统支持 **四种脱敏模式**，适用于不同场景：

| 模式 | 描述 | 适用场景 | 原始数据 | 是否可逆 |
|------|------|----------|----------|----------|
| **静态脱敏** | 创建脱敏后的数据副本 | 开发/测试环境、数据导出 | 不变 | 否 |
| **动态脱敏** | 基于角色的查询时脱敏 | 生产环境权限控制 | 不变 | 否 |
| **原地匿名化** | 永久修改原表数据 | GDPR合规、数据销毁 | 修改 | 不可逆 |
| **泛化脱敏** | 将精确值转换为范围 | 数据分析、统计报表 | 不变 | 否 |

### 如何选择脱敏模式

```
┌─────────────────────────────────────────────────────────────┐
│                    选择哪种脱敏模式？                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  是否需要修改原始数据？                                       │
│  ├── 否 → 需要实时脱敏吗？                                   │
│  │         ├── 是 → 动态脱敏                                │
│  │         └── 否 → 静态脱敏 或 泛化脱敏                     │
│  │                   ├── 保留格式？ → 静态脱敏               │
│  │                   └── 用于分析？ → 泛化脱敏               │
│  │                                                          │
│  └── 是 → 原地匿名化（⚠️ 不可逆！）                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 静态脱敏

静态脱敏创建包含脱敏数据的新表，原始数据保持不变。

#### 创建静态脱敏任务

1. 导航到 **数据脱敏 > 静态脱敏**
2. 点击 **新建任务**
3. 填写表单：
   - **任务名称**: 描述性名称（如"客户数据脱敏"）
   - **数据源**: 选择目标数据库
   - **脱敏模式**: 选择"静态脱敏"
   - **源Schema**: 源模式名（默认: public）
   - **目标Schema**: 目标模式名（默认: public）
4. 点击 **保存**

#### 配置表和字段

1. 在创建的任务上点击 **配置**
2. 点击 **添加表**
3. 输入表配置：
   - **表名**: 此表配置的标识名
   - **源表**: 原始表名
   - **目标表**: 脱敏后的表名（如 `users_masked`）
4. 点击 **从数据库选择** 自动加载字段，或手动添加字段
5. 为每个字段选择脱敏算法并配置参数

#### 执行任务

1. 返回任务列表
2. 在任务上点击 **执行**
3. 系统将：
   - 创建目标表
   - 从源表复制数据
   - 应用脱敏转换
   - 显示执行结果

### 动态脱敏

动态脱敏在查询时基于角色应用脱敏规则，不修改原始数据。不同数据库角色查询同一表时看到不同的数据。

#### 特点：
- 原始数据不变
- 不同角色看到不同数据
- 使用 PostgreSQL SECURITY LABEL 机制
- 查询时实时脱敏
- 适用于生产环境

#### 创建动态脱敏规则

1. 导航到 **数据脱敏 > 动态脱敏**
2. 点击 **新建规则**
3. 配置规则：
   - **规则名称**: 描述性名称
   - **数据源**: 目标数据库
   - **Schema名**: 模式名（默认: public）
   - **表名**: 要应用脱敏的表
   - **被脱敏角色**: 将看到脱敏数据的数据库角色
   - **豁免角色**: 可以查看原始数据的数据库角色
4. 点击 **保存**

#### 添加字段规则

1. 在规则上点击 **配置**
2. 添加字段脱敏规则：
   - **字段名**: 要脱敏的字段
   - **脱敏算法**: 要应用的算法
   - **参数**: 算法参数
3. 点击 **保存**

#### 启用规则

1. 在规则上点击 **启用**
2. 系统将生成并执行 SQL：
   - 在字段上设置 SECURITY LABEL
   - 创建脱敏视图
   - 授予相应权限

#### 生成的 SQL 示例：

```sql
-- 如果不存在则创建被脱敏角色
CREATE ROLE analyst NOINHERIT;

-- 在字段上设置安全标签
SECURITY LABEL FOR anon ON COLUMN public.users.email
  IS 'MASKED WITH FUNCTION anon.fake_email()';

-- 创建脱敏视图
CREATE VIEW public.users_masked AS SELECT * FROM public.users;

-- 授予脱敏视图访问权限
GRANT SELECT ON public.users_masked TO analyst;
```

### 原地匿名化

⚠️ **警告**: 原地匿名化将永久修改原表数据，此操作 **不可逆**。请务必启用备份！

#### 特点：
- 永久修改原表数据
- 可选执行前备份
- 适用于 GDPR 合规
- 数据销毁场景

#### 创建匿名化任务

1. 导航到 **数据脱敏 > 原地匿名化**
2. 点击 **新建任务**
3. 配置：
   - **任务名称**: 描述性名称
   - **数据源**: 目标数据库
   - **Schema名**: 模式名
   - **表名**: 要匿名化的表
   - **执行前备份**: ✓（强烈建议！）
   - **描述**: 任务描述
4. 点击 **保存**

#### 添加字段规则

与静态脱敏相同 - 配置要匿名化的字段和算法。

#### 预览和执行

1. 点击 **预览SQL** 查看生成的 SQL
2. 确认备份表名
3. 点击 **执行** 进行匿名化

#### 生成的 SQL 示例：

```sql
-- 创建备份表
CREATE TABLE public.users_backup_20260304 AS SELECT * FROM public.users;

-- 匿名化字段（永久 UPDATE）
UPDATE public.users SET email = anon.fake_email();
UPDATE public.users SET phone = anon.partial(phone::text, 3, '*', 4);
UPDATE public.users SET salary = anon.noise(salary, 0.15);
```

## 脱敏算法说明

### 算法分类

| 分类 | 描述 | 适用场景 |
|------|------|----------|
| **假数据生成** | 生成逼真的假数据 | 姓名、邮箱、地址 |
| **随机值** | 生成随机数据 | ID、编码 |
| **部分混淆** | 保留部分信息 | 电话号码、身份证 |
| **假名化** | 确定性替换 | 保持一致性的假数据 |
| **哈希** | 单向哈希转换 | 密码、敏感ID |
| **噪声** | 添加随机噪声 | 薪资、年龄 |
| **泛化** | 转换为范围 | 年龄段、薪资范围 |

### 常用算法参考

#### anon.fake_email
生成假但有效的邮箱地址。
```
原始: zhangsan@example.com
脱敏: lauren95@example.com
```

#### anon.fake_first_name
生成假名。
```
原始: 张三
脱敏: Ronnie
```

#### anon.fake_phone_number
生成假电话号码。
```
原始: 13812345678
脱敏: 01-23-45-67-89
```

#### anon.partial
部分混淆 - 保留前缀和后缀。

**参数：**
- `prefix_len`: 保留前缀长度（默认: 2）
- `mask_char`: 掩码字符（默认: '*'）
- `suffix_len`: 保留后缀长度（默认: 2）

```
原始: 13812345678
参数: prefix_len=3, mask_char='*', suffix_len=4
脱敏: 138****5678
```

#### anon.noise
为数值添加随机噪声。

**参数：**
- `ratio`: 噪声比例（如 0.15 表示 ±15%）

```
原始: 15000.00
参数: ratio=0.15
脱敏: 15135.68（在原始值的 ±15% 范围内）
```

#### anon.digest
带自定义盐值的哈希转换。

**参数：**
- `salt`: 哈希盐值
- `algorithm`: 哈希算法（md5, sha1, sha256）

```
原始: 110101199001011234
参数: salt='my_salt', algorithm='sha256'
脱敏: 5d41402abc4b2a76b9719d911017c592...
```

#### anon.hash
使用系统盐值的哈希转换。
```
原始: sensitive_data
脱敏: a1b2c3d4e5f6...
```

## 最佳实践

### 1. 匿名化前务必备份
执行匿名化时启用备份选项，防止意外数据丢失。

### 2. 先用小数据集测试
在生产数据上运行脱敏前，先用小样本测试，确保结果符合预期。

### 3. 选择合适的算法

| 数据类型 | 推荐算法 |
|----------|----------|
| 邮箱 | `anon.fake_email` |
| 电话 | `anon.partial`（保留区号） |
| 姓名 | `anon.fake_first_name`, `anon.fake_last_name` |
| 身份证 | `anon.digest` 或 `anon.hash` |
| 薪资 | `anon.noise` 或泛化 |
| 地址 | `anon.fake_address` |

### 4. 生产环境使用动态脱敏
对于必须保留原始数据的生产环境，使用动态脱敏而不是匿名化。

### 5. 预览生成的 SQL
执行前务必预览生成的 SQL，了解将执行的操作。

## 常见问题

### 连接失败
- 确认数据库主机和端口可访问
- 检查防火墙规则
- 验证用户名和密码
- 确保数据库存在

### 脱敏执行失败
- 检查算法与字段数据类型的兼容性
- 确认目标表不存在（静态脱敏）
- 检查数据库用户是否有必要权限
- 查看执行历史中的错误信息

### 语言切换无效
- 清除浏览器缓存和 Cookie
- 刷新页面
- 检查浏览器 localStorage

### 性能问题
- 大表考虑分批处理
- 适当使用数据库索引
- 在非高峰时段安排脱敏任务
