## 快速开始

### 1. 启动系统

#### 使用 Docker（推荐）
```bash
# 克隆项目
git clone <repository-url>
cd data-masking-tool

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

#### 手动启动
```bash
# 后端
cd backend
python3.10 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 配置数据库等信息
python scripts/init_db.py
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新开终端）
cd frontend
npm install
cp .env.example .env
npm run dev
```

### 2. 访问系统
- 前端地址: http://localhost
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/api/docs

### 3. 登录系统
- 默认账号: `admin`
- 默认密码: `admin123`

> ⚠️ **安全提示**: 首次登录后请立即修改默认密码！

## 数据脱敏

### 1. 创建脱敏任务

1. 点击左侧菜单 **"数据脱敏"**
2. 点击右上角 **"新建任务"**
3. 填写任务信息：
   - 任务名称: 客户信息脱敏
   - 数据源: 选择已配置的数据源
   - 源Schema: public (可选)
   - 目标Schema: public (可选)
   - 调度类型: 手动执行 / 定时调度

### 2. 配置表和字段

点击任务列表中的 **"配置"** 进入任务详情：

#### 添加表
1. 点击 **"添加表"**
2. 填写表名，或从数据源中选择
3. 设置源表和目标表名
4. 点击"确定"

#### 配置字段脱敏
1. 点击表的 **"字段配置"**
2. 点击 **"手动添加"** 或 **"从表选择"**
3. 选择字段和脱敏算法

### 3. 脱敏算法配置

系统提供 72 个内置脱敏算法，分为 8 大类：

| 分类 | 算法示例 | 说明 |
|------|----------|------|
| 假数据生成 | fake_email, fake_name, fake_address | 生成逼真的假数据 |
| 随机值 | random_int, random_string, random_date | 生成随机数据 |
| 部分混淆 | partial, partial_email, partial_phone | 保留部分信息的掩码 |
| 假名化 | pseudo_first_name, pseudo_last_name | 确定性假名替换 |
| 哈希 | digest, hash | 单向哈希转换 |
| 噪声 | add_noise, add_noise_numeric | 添加随机噪声 |
| 泛化 | generalize_date, generalize_number | 将精确值泛化为范围 |
| 条件脱敏 | MASK, REPLACE, NULL, ROUND, OFFSET | 条件脱敏操作符 |

### 4. 执行脱敏任务

1. 在任务列表或详情页点击 **"执行"**
2. 确认后任务开始执行
3. 在 **"执行历史"** 中查看执行状态和结果

### 5. 查看执行详情

任务执行后，可查看详细信息：

1. 点击执行记录可查看详情
2. 信息包括：
   - 执行编号
   - 状态 (SUCCESS/FAILED/RUNNING)
   - 开始时间和结束时间
   - 执行时长 (格式为 "X分Y秒")
   - 总处理记录数
   - 成功/失败记录数
   - 错误信息（如有）

### 6. 生成 SQL

可以为脱敏任务生成 SQL 语句：

1. 在任务详情页点击 **"生成SQL"**
2. 查看生成的 SQL 语句
3. SQL 包括：
   - DROP TABLE (如果目标表存在)
   - CREATE TABLE (复制表结构)
   - INSERT (复制数据)
   - UPDATE (应用脱敏)

### 7. 使用 HashData Anon 插件

系统底层集成了 HashData Lightning 的 Anon 插件，提供高性能脱敏：

```sql
-- Anon 插件使用示例
SELECT anon.partial('13812345678', 3, 4);  -- 138****5678
SELECT anon.digest('secret', 'sha256');      -- 哈希脱敏
SELECT anon.random_first_name();               -- 随机名
```

### 8. 数值字段注意事项

使用 **REPLACE** 算法处理数值字段时：

- 确保替换值在字段的精度范围内
- 例如 `numeric(7,2)` 最大值为 99999.99
- 使用如 `99.99` 而非 `123123` 的值
- 系统会自动检测数值类型并正确格式化

## 脱敏算法说明

### 1. 掩码脱敏 (MASK)

适用于手机号、身份证号、银行卡号等需要保留部分信息的字段。

**参数**:
- `prefix_length`: 前缀保留长度，默认3
- `suffix_length`: 后缀保留长度，默认4
- `mask_char`: 掩码字符，默认*

**示例**:
```
手机号: 13812345678 → 138****5678
身份证: 110101199001011234 → 110***********1234
```

### 2. 哈希脱敏 (HASH)

使用 SHA-256 算法进行不可逆哈希，适用于唯一标识。

**参数**:
- `salt`: 盐值（可选，增加安全性）

**示例**:
```
原始值: abc123
哈希值: a665a45920422f9d417e4866ef6e32b7...
```

### 3. 替换脱敏 (REPLACE)

使用固定值替换原始数据。

**参数**:
- `replacement`: 替换值

**注意**: 对于数值字段，确保替换值在字段的精度范围内。

**示例**:
```
原始值: 张三
替换值: ***

对于 numeric(7,2): 使用 99.99 而非 123123
```

### 4. 空值脱敏 (NULL)

将字段值设为 NULL。

### 5. 取整脱敏 (ROUND)

对数值进行取整，适用于金额、年龄等。

**参数**:
- `precision`: 精度，负数表示10的倍数

**示例**:
```
precision=-3: 12345.67 → 12000
precision=2: 12.345 → 12.35
```

### 6. 偏移脱敏 (OFFSET)

对数值进行固定偏移，适用于需要保留相对关系的场景。

**参数**:
- `offset`: 偏移量
- `min_value`: 最小值（可选）
- `max_value`: 最大值（可选）

**示例**:
```
offset=5: 30 → 35
```

### 7. 格式保持脱敏 (PRESERVATION)

保持数据格式的同时进行随机化，如姓名替换为随机姓名、地址替换为随机地址。

### 8. 假数据生成 (FAKE)

生成逼真的假数据：
- `anon.fake_email()` - 随机邮箱
- `anon.fake_first_name()` - 随机名
- `anon.fake_last_name()` - 随机姓
- `anon.fake_address()` - 随机地址
- `anon.fake_city()` - 随机城市
- `anon.fake_country()` - 随机国家

## 常见问题

### Q: 测试数据源连接失败怎么办？
A: 检查以下几点：
- 网络是否通：`telnet host port`
- 用户名密码是否正确
- 数据库是否允许远程连接（检查 pg_hba.conf）
- 防火墙是否开放端口

### Q: 脱敏任务执行很慢怎么办？
A: 优化建议：
- 使用 HashData MPP 数据库利用并行处理
- 分批处理大表
- 避开业务高峰期执行
- 检查 Anon 插件是否启用

### Q: 如何回滚脱敏操作？
A: 建议：
- 脱敏前先备份数据
- 使用目标表而非覆盖源表
- 保留源数据直到验证通过

### Q: 支持哪些数据库类型？
A: 当前支持：
- HashData Lightning (MPP)
- PostgreSQL
- MySQL
- Oracle
- GoldenDB
- 达梦

### Q: 如何使用定时调度？
A: 配置 Cron 表达式即可：
- `0 0 2 * * ?` 每天凌晨2点
- `0 0 2 * * 0` 每周日凌晨2点
- `0 0 2 1 * ?` 每月1号凌晨2点

### Q: 为什么 REPLACE 算法在数值字段上失败？
A: 确保替换值在字段的数值精度范围内：
- 检查字段定义（如 `numeric(7,2)` 表示最大约 99999.99）
- 使用更小的符合精度的值
- 系统会自动检测数值类型，但值必须有效
