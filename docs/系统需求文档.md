## 1. 项目概述

### 1.1 项目背景
本机构作为数据主线主要存储和供数系统依托MPP数据库，长期面临庞大的数据脱敏和翻数需求。当前研发测试环境每年多次截面数据导出，涉及233个应用、约100T脱敏数据，MPP数据库暂无成熟脱敏工具，依赖人工识别和手写代码，平均周期7天左右，在数据安全管控和研发效率上存在突出卡点。

为满足测试环境用数需求，加强数据脱敏环节操作管理，拟通过采购第三方脱敏软件，获取专业、可靠、高效的数据脱敏技术能力，完成数据条线等领域的主题分析，通过平台搭建和数据实施促进企业数字化转型。

### 1.2 项目目标
通过POC测试验证厂商技术实力，最终选定合作伙伴，搭建统一的血缘分析、MPP数据脱敏、翻数工具平台，解决MPP数据库无工具支持的数据脱敏和翻数问题，提升研发效率，防范数据外泄风险，保障信息安全。

### 1.3 项目范围
| 模块 | 说明 |
|------|------|
| 数据血缘分析 | 数据源登记、血缘关系可视化 |
| MPP数据脱敏 | 基于HashData Lightning + Anon插件的数据脱敏 |
| 翻数工具 | 高效的数据翻数操作 |
| 安全管理 | 用户、角色、权限管理 |

## 3. 非功能需求规格

### 3.1 性能需求 (NFR-001)

| 指标 | 要求 |
|------|------|
| 数据处理规模 | 支持约100T数据脱敏处理 |
| 处理周期 | 脱敏周期从人工7天缩短至工具自动化处理 |
| 并发用户 | 支持至少50个并发用户 |
| 页面响应时间 | 一般页面<2秒，复杂查询<5秒 |
| 任务调度 | 支持至少100个并发脱敏任务 |

### 3.2 兼容性需求 (NFR-002)

| 类别 | 要求 |
|------|------|
| 数据库 | HashData Lightning、Oracle、GoldenDB、MySQL、达梦 |
| 浏览器 | Chrome、Firefox、Edge、Safari（最新两个版本） |
| 操作系统 | 服务端：Linux（CentOS 7+、Ubuntu 18+）<br>客户端：Windows、macOS |

### 3.3 安全性需求 (NFR-003)

| 类别 | 要求 |
|------|------|
| 数据传输 | HTTPS/TLS加密传输 |
| 数据存储 | 敏感配置加密存储 |
| 密码安全 | 密码加盐哈希存储，支持密码复杂度策略 |
| 审计追踪 | 完整的操作审计日志 |
| 权限隔离 | 严格的功能和数据权限隔离 |
| 合规性 | 符合《本机构信息及信息系统安全管理办法》 |

### 3.4 可靠性需求 (NFR-004)

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥99.5% |
| 数据备份 | 支持配置和任务数据备份 |
| 故障恢复 | 任务失败支持断点续传/重试 |

### 3.5 可扩展性需求 (NFR-005)

| 方面 | 要求 |
|------|------|
| 数据源扩展 | 支持新数据源类型扩展 |
| 脱敏算法扩展 | 支持自定义脱敏算法扩展 |
| 部署方式 | 支持容器化部署 |

### 3.6 易用性需求 (NFR-006)

| 方面 | 要求 |

## 7. 安全要求

### 7.1 认证与授权
1. 所有API接口需通过认证才能访问（登录接口除外）
2. 采用JWT Token进行身份认证
3. Token设置合理的过期时间，支持刷新机制
4. 基于RBAC模型进行权限控制
5. 敏感操作需验证用户权限

### 7.2 数据安全
1. 数据库密码等敏感配置加密存储（使用AES-256）
2. 密码使用bcrypt/Argon2加盐哈希存储
3. HTTPS/TLS加密传输
4. 敏感数据在日志中脱敏显示
5. 支持数据备份与恢复

### 7.3 审计要求
1. 记录用户登录/登出行为
2. 记录关键业务操作（创建、修改、删除等）
3. 记录数据访问行为
4. 日志至少保留6个月
5. 日志不可篡改

### 7.4 合规性
1. 符合《本机构信息及信息系统安全管理办法》
2. 符合国家网络安全等级保护要求
3. 符合个人信息保护相关法律法规
