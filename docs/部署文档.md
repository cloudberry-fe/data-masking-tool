## 环境要求

### 服务器配置
- **操作系统**: Linux (CentOS 7+, Ubuntu 18.04+)
- **CPU**: 4核及以上
- **内存**: 8GB及以上
- **磁盘**: 100GB及以上

### 软件依赖
- **Python**: 3.10+
- **Node.js**: 18+
- **PostgreSQL**: 14+ (用于存储元数据)
- **Redis**: 6+ (用于任务队列和缓存)
- **HashData Lightning**: (MPP数据库，用于数据脱敏)

## 后端部署

### 1. 克隆项目
```bash
cd /opt
git clone <repository-url> data-masking-tool
cd data-masking-tool/backend
```

### 2. 创建Python虚拟环境
```bash
python3.10 -m venv venv
source venv/bin/activate
```

### 3. 安装依赖
```bash
pip install -r requirements.txt
```

### 4. 配置环境变量
```bash
cp .env.example .env
vim .env
```

需要配置的主要参数：

```env
# 应用配置
APP_NAME=数据脱敏系统
APP_ENV=production
APP_DEBUG=false
APP_HOST=0.0.0.0
APP_PORT=8000

# 数据库配置
DATABASE_URL=postgresql://username:password@localhost:5432/data_masking

# Redis配置
REDIS_URL=redis://localhost:6379/0

# JWT配置（生产环境请使用强密钥）
JWT_SECRET_KEY=your-very-strong-secret-key-change-this-in-production
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=1440

# 加密配置（32字节密钥）
ENCRYPTION_KEY=your-32-byte-encryption-key-here

# HashData Lightning 配置
HASHDATA_HOST=hashdata-host
HASHDATA_PORT=5432
HASHDATA_DATABASE=hashdata
HASHDATA_USERNAME=gpadmin
HASHDATA_PASSWORD=your-password
```

### 5. 初始化数据库
```bash
# 创建数据库
psql -U postgres
CREATE DATABASE data_masking;
\q

# 初始化表和数据
python scripts/init_db.py
```

### 6. 启动服务
```bash
# 开发环境
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 生产环境
pip install gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000 app.main:app
```

### 7. 使用 systemd 管理服务
创建 `/etc/systemd/system/data-masking-backend.service`:

```ini
[Unit]
Description=Data Masking Backend Service
After=network.target postgresql.service redis.service

[Service]
Type=notify
User=app
WorkingDirectory=/opt/data-masking-tool/backend
Environment="PATH=/opt/data-masking-tool/backend/venv/bin"
ExecStart=/opt/data-masking-tool/backend/venv/bin/gunicorn -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000 app.main:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
systemctl daemon-reload
systemctl enable data-masking-backend
systemctl start data-masking-backend
systemctl status data-masking-backend
```

## 前端部署

### 1. 安装依赖
```bash
cd /opt/data-masking-tool/frontend
npm install
```

### 2. 配置环境变量
```bash
cp .env.example .env
vim .env
```

```env
VITE_API_BASE_URL=http://your-backend-host:8000/api/v1
```

### 3. 构建前端
```bash
npm run build
```

### 4. 使用 Nginx 部署
将 `dist` 目录的内容部署到 Nginx：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/data-masking-tool/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 启用gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript;
}
```

## HashData Lightning + Anon 插件配置

### 1. 安装 Anon 插件
```sql
-- 连接到 HashData Lightning
psql -h hashdata-host -p 5432 -U gpadmin -d hashdata

-- 创建扩展
CREATE EXTENSION IF NOT EXISTS anon CASCADE;

-- 验证安装
SELECT extversion FROM pg_extension WHERE extname = 'anon';
```

### 2. 配置 Anon 插件
```sql
-- 启用动态脱敏（可选）
ALTER DATABASE hashdata SET anon.mask_function = 'anon.mask';

-- 查看可用的脱敏函数
\df anon.*
```

### 3. 常用 Anon 脱敏函数示例
```sql
-- 掩码脱敏
SELECT anon.partial_email('user@example.com');
SELECT anon.partial('13812345678', 3, 4);

-- 哈希脱敏
SELECT anon.digest('sensitive data', 'sha256');

-- 随机化
SELECT anon.random_first_name();
SELECT anon.random_last_name();
SELECT anon.random_date_between('1970-01-01'::date, '2000-01-01'::date);

-- 噪声添加
SELECT anon.add_noise(1000, 0.1);
```

## Docker 部署

### 使用 docker-compose

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:14-alpine
    container_name: data-masking-db
    environment:
      POSTGRES_DB: data_masking
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: data-masking-redis
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: data-masking-backend
    environment:
      DATABASE_URL: postgresql://postgres:postgres@postgres:5432/data_masking
      REDIS_URL: redis://redis:6379/0
      JWT_SECRET_KEY: your-secret-key
      ENCRYPTION_KEY: your-32-byte-encryption-key
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started
    ports:
      - "8000:8000"
    volumes:
      - ./logs:/app/logs
      - ./uploads:/app/uploads

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: data-masking-frontend
    depends_on:
      - backend
    ports:
      - "80:80"

volumes:
  postgres_data:
  redis_data:
```

### 后端 Dockerfile (`backend/Dockerfile`)

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# 安装Python依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 创建必要的目录
RUN mkdir -p logs uploads

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["gunicorn", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "-b", "0.0.0.0:8000", "app.main:app"]
```

### 前端 Dockerfile (`frontend/Dockerfile`)

```dockerfile
# 构建阶段
FROM node:18-alpine as builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

# 运行阶段
FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 前端 Nginx 配置 (`frontend/nginx.conf`)

```nginx
events {
    worker_connections 1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    server {
        listen 80;
        server_name localhost;
        root /usr/share/nginx/html;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
        }

        location /api {
            proxy_pass http://backend:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        gzip on;
        gzip_vary on;
        gzip_min_length 1024;
        gzip_types text/plain text/css text/xml text/javascript application/json application/javascript;
    }
}
```

### 启动容器
```bash
docker-compose up -d

# 查看日志
docker-compose logs -f backend

# 初始化数据库
docker-compose exec backend python scripts/init_db.py
```

## 常见问题

### 1. 后端启动失败
**问题**: `ModuleNotFoundError: No module named 'app'`

**解决**: 确保在项目根目录启动，或设置 `PYTHONPATH`:
```bash
export PYTHONPATH=/opt/data-masking-tool/backend
```

### 2. 数据库连接失败
**问题**: `connection to server at "localhost" failed`

**解决**: 检查 `.env` 中的 `DATABASE_URL` 配置，确认 PostgreSQL 服务已启动。

### 3. HashData 连接测试失败
**问题**: 测试 HashData 连接时提示失败

**解决**:
- 检查网络连通性: `telnet hashdata-host 5432`
- 确认用户名密码正确
- 检查 HashData 的 `pg_hba.conf` 配置

### 4. Anon 插件不可用
**问题**: 提示 `function anon.partial does not exist`

**解决**: 在 HashData 数据库中执行:
```sql
CREATE EXTENSION IF NOT EXISTS anon CASCADE;
```

### 5. 前端页面空白
**问题**: 打开页面显示空白

**解决**:
- 检查浏览器控制台的错误信息
- 确认 `VITE_API_BASE_URL` 配置正确
- 检查后端服务是否正常运行

### 6. 脱敏任务返回零条记录
**问题**: 任务执行成功但显示处理0条记录

**解决**:
- 检查源表是否存在且有数据
- 确认 schema 名称配置正确（source_schema/target_schema）
- 检查表是否已启用
- 查看后端日志中的 SQL 执行警告

### 7. 数值字段溢出错误
**问题**: 使用 REPLACE 算法时提示 `numeric field overflow`

**解决**:
- 确保替换值在字段的数值精度范围内
- 对于 `numeric(7,2)`，最大值约为 99999.99
- 使用合适的值如 `99.99` 而非 `123123`
