## 脱敏算法库

系统预置了 10 种脱敏算法：

| 算法编码 | 算法名称 | 说明 |
|----------|----------|------|
| MASK | 掩码脱敏 | 保留前缀后缀，中间用*替换 |
| HASH | 哈希脱敏 | SHA-256 不可逆哈希 |
| REPLACE | 替换脱敏 | 固定值替换 |
| NULL | 空值脱敏 | 置为 NULL |
| ROUND | 取整脱敏 | 数值范围取整 |
| OFFSET | 偏移脱敏 | 数值固定偏移 |
| SHUFFLE | 洗牌脱敏 | 列内数据随机打乱 |
| SUBSTITUTION | 字典替换 | 字典数据替换 |
| PRESERVATION | 格式保持 | 保持格式的脱敏 |
| ENCRYPT | 加密脱敏 | 可逆加密 |

## HashData Lightning + Anon 集成

### 核心特性
1. **高性能 MPP 架构**: 利用 HashData Lightning 的并行处理能力
2. **Anon 插件原生支持**: 直接使用 PostgreSQL Anon 插件的脱敏函数
3. **SQL 生成**: 自动生成脱敏 SQL，支持 CREATE TABLE AS 和 INSERT INTO 两种方式
4. **批量处理**: 支持大批量数据高效处理

### 使用方式
```python
from app.utils.hashdata_anon import HashDataAnonManager, MaskingTableConfig, MaskingColumnConfig

# 初始化管理器
manager = HashDataAnonManager(datasource_config)

# 构建脱敏配置
table_config = MaskingTableConfig(
    source_table="customer",
    target_table="customer_masked",
    columns=[
        MaskingColumnConfig("phone", "MASK", {"prefix_length": 3, "suffix_length": 4}),
        MaskingColumnConfig("id_card", "HASH"),
        MaskingColumnConfig("name", "PRESERVATION"),
    ]
)

# 执行脱敏
result = manager.execute_masking(table_config)
```
